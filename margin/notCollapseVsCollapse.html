Margin Collapse vs No Collapse

The Key Difference:

  • Collapse: Margins "merge" together, space gets smaller than expected
  • No Collapse: Margins work as expected, adding up to create the full space

❌ MARGINS COLLAPSE

Child Element
margin: 40px 0
⚠️ Margin "escapes" the parent!
The child's margin pushes the ENTIRE parent down

✅ MARGINS DON'T COLLAPSE

Child Element
margin: 40px 0
✅ Margin stays inside parent!
Creates actual space inside the container

What's Happening?

❌ When Margins Collapse:

.parent { /* No padding/border */ } .child { margin-top: 40px; }

Result: The child's margin "escapes" and pushes the entire parent container down by 40px, instead of creating space inside the parent.

✅ When Margins Don't Collapse:

.parent { padding: 1px; /* Prevents collapse */ } .child { margin-top: 40px; }

Result: The child's margin creates actual 40px space inside the parent container, which is what you'd normally expect.

How to Prevent Margin Collapse:

  • Add padding to the parent (even 1px works!)
  • Add border to the parent
  • Use overflow: hidden on the parent
  • Use display: flex or display: grid on the parent
  • Use position: absolute on the child
